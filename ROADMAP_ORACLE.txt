2) OBJECTIF PRODUIT COURT TERME
================================================================================

Livrer un MVP 1v1 en ligne stable, équitable et testable:
- même état de combat sur les deux clients
- résultat de match fiable en cas d’erreur réseau
- boucle de jeu fluide (passif -> placement -> combat -> résultat)

Hors scope MVP:
- 2v2, 3v3
- MMR avancé et saisons
- replay/spectateur

================================================================================
3) PLAN D’EXÉCUTION EN 3 PHASES
================================================================================

PHASE A — Stabilisation de la vérité réseau (priorité absolue)
---------------------------------------------------------------
1. Passifs synchronisés:
   - tirage d’offre passifs par seed partagé ou autorité MasterClient
   - validation du choix passif via RPC/événement réseau
2. Timeouts et fin de tour:
   - une seule autorité réseau pour expirer un tour
3. Déconnexion:
   - règle explicite (forfait, victoire adverse, retour hub)
4. Migration MasterClient:
   - stratégie de reprise des rôles critiques sans incohérence

Critères de sortie phase A:
- 10+ matchs test 1v1 sans désynchronisation visible
- pas de blocage de combat après déconnexion d’un joueur

PHASE B — Durcissement gameplay + UX de match
---------------------------------------------
1. Validation renforcée des actions réseau (cast/move/end turn).
2. Messages UI clairs:
   - état de connexion
   - attente adversaire
   - fin de match (victoire/forfait)
3. Test de robustesse:
   - latence simulée
   - pertes temporaires de connexion
   - rejoin/retry selon stratégie retenue

Critères de sortie phase B:
- tests passants sur scénarios nominal + perturbés
- expérience utilisateur compréhensible sans debug console

PHASE C — Polish MVP et release candidate
-----------------------------------------
1. Harmonisation HUD et feedback combat.
2. Audio/VFX minimum viable.
3. Nettoyage final des logs/debug temporaires.
4. Guide d’exploitation:
   - comment lancer un test 2 joueurs
   - comment diagnostiquer les incidents réseau courants

Critères de sortie phase C:
- checklist MVP validée
- release candidate interne jouable end-to-end

================================================================================
4) DÉFINITION OF DONE (MVP 1V1 ONLINE)
================================================================================

Le MVP est considéré prêt si:
- sélection passif synchrone et validée des deux côtés
- placement, déplacement, sort, fin de tour synchronisés sans divergence
- gestion déconnexion claire et reproductible
- résultat de match déterministe (pas de double-victoire ni blocage)
- parcours MainMenu -> Hub -> Match -> Combat -> Résultat fonctionnel
